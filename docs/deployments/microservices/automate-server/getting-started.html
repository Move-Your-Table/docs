Getting started

Requirements‚Äã

  • Rust

    üí° While this is listed within the requirements, you'll only need this if you choose to run your code locally. Docker takes care of Rust!

  • Docker Desktop

Setup‚Äã

‚Äº When within staging & production, the image will need to be rebuild when code changes happen. For this, add the --build argument behind the given docker-compose commands

Development‚Äã

  1. Clone the project to your local machine using your favorite CLI & git clone
  2. Copy the .env.example, rename it to .env and fill it in according to the .env configuration section.
  3. Run the actual project
  • If you want to develop within the container, you'll need to use the following command:
$ docker-compose --profile dev-container up

üö® The first build might take a while. Rust takes a while to build.

  • If you want to run your build locally, you'll need to use the following command:
$ docker-compose --profile dev-local up
  1. When you want to get the containers offline, enter the following command:
# If you were developing locally
$ docker-compose --profile dev-local down

# If you were developing within containers
$ docker-compose --profile dev-container down

Adding other services‚Äã

Because Gateway, Automation API and Back-End share the same custom network, you can build these and get the production server up & running.

  • For Back-End, clone the project, navigate to the root of the project and perform the following command:

    $ docker-compose up -d nestjs_dev
    $ docker-compose up -d mongodb_dev

    ‚ö†Ô∏è Currently, the GraphQL implementation in the production container seems to be broken, so we suggest using the development version for now.

  • For Gateway, clone the project, navigate to the root of the project and perform the following command:

    $ docker-compose up tsed_prod -d
