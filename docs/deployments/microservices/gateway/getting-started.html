Requirements‚Äã

  • Docker (Desktop)

Development Setup‚Äã

  1. Clone this project using the following command:
    $ git clone git@git.ti.howest.be:TI/2021-2022/s5/trending-topics/projects/hybrid-work1/gateway.git
  2. Run npm install:
    $ yarn install
  3. Copy the .env.example file, rename it to .env and fill it in according to the .env configuration section
  4. Now, run the following command:
    $ yarn start:container

    ‚è± This will take a while the first time. Go grab a ‚òï while waiting.

‚úÖ Done! Normally the application should be available on localhost:<TSED_PORT> and should be able to contact other services.

Adding other services‚Äã

Because Gateway, Automation API and Back-End share the same custom network, you can build these and get the production server up & running.

  • For Back-End, clone the project, navigate to the root of the project and perform the following command:

    $ yarn start:container:dev

    ‚ö†Ô∏è Currently, the GraphQL implementation in the production container seems to be broken, so we suggest using the development version for now.

  • For Automation API, clone the project, navigate to the root of the project and perform the following command:

    $ docker-compose up rust_prod -d

Available commands‚Äã

CommandExplication
yarn startStarts a development server locally
yarn start:prodStarts a production-ready server locally
yarn start:containerStart a fully containerized development server with hot-reloading capabilities.
yarn start:container:prodStart a fully containerized production server
